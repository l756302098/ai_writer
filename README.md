# AI 写作工作室

基于 React + Tiptap + DeepSeek API 的本地写作应用，数据全部存储在浏览器 IndexedDB，无需后端。

---

## 技术栈

- **前端框架**：React 18 + TypeScript + Vite
- **富文本编辑器**：Tiptap
- **样式**：TailwindCSS
- **本地存储**：Dexie（IndexedDB 封装）
- **AI 接入**：DeepSeek API

---

## 快速开始

### 1. 安装依赖

```bash
npm install
```

### 2. 启动开发服务器

```bash
npm run dev
```

浏览器访问 [http://localhost:5173](http://localhost:5173)

### 3. 构建生产版本

```bash
npm run build
```

构建产物在 `dist/` 目录，可直接双击 `dist/index.html` 打开（无需服务器）。

## 使用教程

### 第一步：配置 API Key

1. 打开应用后，点击左下角 **⚙ 设置**
2. 在 **DeepSeek API Key** 输入框中填入你的 Key（格式：`sk-...`）
   - 前往 [DeepSeek 开放平台](https://platform.deepseek.com) 注册并获取 API Key
3. 选择模型：
   - `deepseek-v4-flash`：通用写作对话，响应快
   - `deepseek-v4-pro`：深度推理，适合复杂改写任务
4. 选择写作风格偏好（影响 AI 的回复风格）
5. 点击 **保存**

> API Key 仅存储在本地浏览器 IndexedDB，不会上传到任何服务器。

---

### 第二步：创建章节

- 点击左侧侧边栏顶部的 **+ 新建章节**
- 在顶部标题栏输入章节名称
- 当章节标题为"未命名章节"时，编辑器会自动以内容首行（前 30 字）作为标题

---

### 第三步：写作

在主编辑区直接输入内容，支持以下格式：

| 操作 | 说明 |
|------|------|
| 工具栏按钮 | 加粗、斜体、H1/H2 标题、列表、引用 |
| 左侧列表 | 点击切换章节，悬停显示删除按钮 |
| 顶部搜索框 | 按标题或内容关键词筛选章节 |
| 字数统计 | 章节标题下方实时显示字数和修改时间 |

---

### 第四步：使用 AI 助手

#### 方式一：快捷键呼出

1. 在编辑器中**选中一段文字**
2. 按 `Ctrl+J`，AI 面板自动弹出，并带入选中内容
3. 在输入框描述需求，按 `Enter` 发送

#### 方式二：直接打开

1. 按 `Ctrl+J`（无需选中文字）
2. 在输入框输入任意写作需求

#### AI 面板操作

- `Enter`：发送消息
- `Shift+Enter`：输入框内换行
- **将最新回复插入编辑器**：点击对话下方的链接，将 AI 回复追加到当前章节末尾
- **✕**：关闭面板并清空本轮对话

#### 常用需求示例

```
帮我润色这段文字，保持原意但更流畅
续写这个场景，保持当前人物性格
给这段对话加上动作描写
把这段改写成第一人称
帮我检查语病和错别字
根据以下大纲，帮我扩写第二节
```

## 数据管理

所有数据（章节内容、设置、API Key）均存储在浏览器 IndexedDB 数据库 `WritingStudioDB` 中。

- **清除数据**：在浏览器开发者工具 → Application → IndexedDB → 删除 `WritingStudioDB`
- **数据迁移**：目前不支持导出，如需备份请手动复制章节内容

## UI 功能设计

整个应用采用**三栏布局**：左侧为大纲导航栏，中间为主编辑区，右侧为 AI 助手面板。以下为各核心功能的界面设计描述。

---

### 一、故事大纲生成与管理

**入口位置**：左侧大纲导航栏顶部，设有一个「生成大纲」按钮和一行描述输入框。

**交互流程**：

1. 用户在输入框中用自然语言描述故事构想（如"我想写一个关于星际殖民时代的外交官故事，主角是一位被派往异星文明的人类大使"），点击「生成大纲」或按 `Ctrl+Enter` 发送。
2. AI 根据描述生成大纲，以**二级目录树**的形式呈现在左侧导航栏中：
   - **一级节点**为卷/章（如"第一章：启程"），字体加粗，左侧有展开/折叠箭头。
   - **二级节点**为节/小节（如"1.1 接到任命"、"1.2 星际飞船上的邂逅"），缩进显示，字体略小。
   - 每个二级节点可点击跳转，主编辑区自动定位到对应章节内容。
3. 大纲节点支持**拖拽排序**，用户可以手动调整章节顺序。
4. 每个节点右侧悬停时显示操作按钮：**重命名**（铅笔图标）、**删除**（垃圾桶图标）、**在此之后新增**（加号图标）。

**动态更新机制**：

- 当用户修改描述文本（如补充新的情节设定、调整故事走向）并再次触发生成时，AI 不会全量覆盖原有大纲，而是**增量更新**：
  - 新增的章节/小节以**绿色高亮**标记，持续 3 秒后渐隐。
  - 被修改的节点标题以**橙色高亮**标记。
  - 被删除的节点先以**红色高亮 + 删除线**标记，用户可点击「撤销删除」恢复，10 秒后自动清除。
- 用户可在任意节点上右键选择「从此节点重新生成」，仅更新该节点及其子树，不影响其他部分。

**界面示意**：

```
┌─ 大纲导航栏 ──────────────────┐
│ 📝 [描述输入框..............]  │
│ [生成大纲]                     │
│ ─────────────────────────────  │
│ 📖 第一章：启程          ▼    │
│    📄 1.1 接到任命             │
│    📄 1.2 星际飞船上的邂逅     │
│    📄 1.3 抵达异星首都         │
│ 📖 第二章：漩涡中心      ▼    │
│    📄 2.1 第一次外交谈判       │
│    📄 2.2 暗流涌动             │
│ 📖 第三章：转折          ▶    │
│ ─────────────────────────────  │
│ [+ 新建章节]                   │
└────────────────────────────────┘
```

### 二、故事背景与人物设定

**入口位置**：左侧大纲导航栏下方设有**标签页切换**——「大纲」/「世界观」/「人物」。

#### 2.1 世界观设定（「世界观」标签页）

- 顶部为一个**富文本编辑区**，用户可自由撰写故事背景（时代背景、地理环境、社会制度、科技水平、魔法体系等）。
- 编辑区下方有一个「AI 生成世界观」按钮，输入关键词（如"赛博朋克 + 东方玄幻"），AI 自动生成结构化的世界观描述，填充到编辑区。
- 世界观内容随写作进度**持续可编辑**，所有修改自动保存到 IndexedDB。

#### 2.2 人物设定（「人物」标签页）

- 以**人物卡片列表**的形式展示所有已创建角色。
- 每张卡片包含：

| 字段 | 说明 |
|------|------|
| 姓名 | 角色名称 |
| 身份/职业 | 在故事中的定位 |
| 外貌描写 | 自由文本 |
| 性格特征 | 标签式（如「冷静」「多疑」「幽默」）+ 详细描述 |
| 心理状态 | 当前故事节点下的心理活动描述 |
| 动机/目标 | 角色的核心驱动力 |
| 与其他角色关系 | 关联角色名 + 关系描述 |

- 卡片可**展开/折叠**，展开时显示全部字段，折叠时仅显示姓名和身份。
- 右上角有「+ 新增人物」按钮，点击弹出空白卡片供填写；也可点击「AI 生成人物」，输入简短描述后由 AI 补全人物设定。

**人物动态演变**：

- 在主编辑区写作过程中，用户可以**选中某段文字**，右键选择「关联人物 → [角色名]」，将该段内容与指定角色关联。
- 当用户在 AI 面板中请求「根据当前故事进展，更新 XXX 的性格和心理状态」时，AI 会**扫描所有关联该角色的段落**，基于上下文自动更新人物卡片中的**性格特征**和**心理状态**字段。
- 更新后的内容以**蓝色标记**高亮显示在卡片中，用户可选择「采纳」或「放弃」。
- 每次更新会记录在卡片底部的**演变时间线**中（如"第五章：经历了背叛事件后，信任感降低，变得多疑"），形成完整的人物弧光追踪。

**界面示意**：

```
┌─ 人物标签页 ──────────────────┐
│ [+ 新增人物] [AI 生成人物]     │
│ ─────────────────────────────  │

### 三、上下文感知的章节润色

**入口位置**：AI 面板（右侧）中的「润色」快捷操作区。

**交互流程**：

1. 用户在主编辑区打开待润色的章节，点击 AI 面板顶部的「✨ 润色当前章节」按钮（或选中部分文字后按 `Ctrl+J` 仅润色选中内容）。
2. 润色请求自动携带以下**上下文信息**发送给 AI：
   - **当前章节完整内容**（或选中的文字片段）
   - **所在章节的大纲节点**（如"2.1 第一次外交谈判"）及其兄弟节点标题，帮助 AI 理解当前章节在整体结构中的位置
   - **关联人物卡片**：当前章节关联的所有角色的性格、心理状态、说话风格
   - **世界观设定摘要**：当前故事背景的关键设定点
   - **用户自定义润色指令**：如"让对话更加正式"、"增强紧张氛围"、"减少副词使用"
3. AI 返回润色后的文本，在 AI 面板中以**对比视图**展示：
   - 左侧为原文，右侧为润色版
   - 差异处以**行级高亮**标出（新增绿色、删除红色、修改橙色）
4. 用户可选择：
   - **「全部替换」**：将润色版完整替换原文
   - **「逐处确认」**：逐条差异点供用户确认，类似于代码审查中的 diff 逐块接受
   - **「追加到下方」**：将润色版作为新版本追加，保留原文
   - **「放弃」**：丢弃润色结果

**润色指令模板**：

在 AI 面板的润色模式下，提供一组快捷指令模板供用户一键选用：

| 模板 | 说明 |
|------|------|
| 语言流畅 | 优化句式结构，消除语病和冗余表达 |
| 增强描写 | 丰富场景细节、动作描写和感官描述 |
| 人物对话 | 使对话更符合各角色的性格和说话风格 |
| 节奏控制 | 调整段落长短和叙事节奏 |
| 语病检查 | 专注查找语法错误、错别字和标点问题 |
| 自定义… | 用户输入具体润色要求 |

**界面示意**：

```

│ [✨ 润色] [💬 对话] [📋 大纲]  │

│ 润色范围：                      │
│  ● 当前章节（第二章 第一节）     │
│  ○ 选中文字                     │
